---
title: Mode
description: Configure webpack's built-in optimizations based on the build environment
---

Providing the `mode` configuration option tells webpack to use its built-in optimizations accordingly. It can be set to `'production'`, `'development'`, or `'none'`.

## Configuration

```javascript webpack.config.js
module.exports = {
  mode: 'production'
};
```

Or from CLI:
```bash
webpack --mode=production
```

## Mode Options

<Tabs>
  <Tab title="Development">
    ```javascript
    module.exports = {
      mode: 'development'
    };
    ```
    
    Sets `process.env.NODE_ENV` to `'development'` and enables:
    - Fast incremental builds
    - Readable output
    - Detailed error messages
    - Source maps
    - No minification
  </Tab>
  
  <Tab title="Production">
    ```javascript
    module.exports = {
      mode: 'production'
    };
    ```
    
    Sets `process.env.NODE_ENV` to `'production'` and enables:
    - Minification
    - Tree shaking
    - Scope hoisting
    - Performance optimizations
    - Smaller bundle sizes
  </Tab>
  
  <Tab title="None">
    ```javascript
    module.exports = {
      mode: 'none'
    };
    ```
    
    Opts out of all default optimizations.
  </Tab>
</Tabs>

## Loader Context

From `NormalModule.js`, the mode is available in loader context:

```javascript NormalModule.js
const loaderContext = {
  mode: options.mode || 'production',
  webpack: true,
  sourceMap: Boolean(this.useSourceMap),
  // ... other properties
};
```

Loaders can access the mode:
```javascript my-loader.js
module.exports = function(source) {
  const isDevelopment = this.mode === 'development';
  
  if (isDevelopment) {
    // Add development-specific code
    return addDebugInfo(source);
  }
  
  return source;
};
```

## Development Mode Details

When mode is set to `'development'`, webpack applies:

```javascript
// Equivalent configuration
module.exports = {
  mode: 'development',
  devtool: 'eval',
  cache: true,
  performance: {
    hints: false
  },
  output: {
    pathinfo: true
  },
  optimization: {
    moduleIds: 'named',
    chunkIds: 'named',
    mangleExports: false,
    nodeEnv: 'development',
    flagIncludedChunks: false,
    occurrenceOrder: false,
    concatenateModules: false,
    splitChunks: {
      hidePathInfo: false,
      minSize: 10000,
      maxAsyncRequests: Infinity,
      maxInitialRequests: Infinity
    },
    emitOnErrors: true,
    checkWasmTypes: false,
    minimize: false,
    removeAvailableModules: false
  },
  plugins: [
    new webpack.DefinePlugin({ 'process.env.NODE_ENV': JSON.stringify('development') })
  ]
};
```

## Production Mode Details

When mode is set to `'production'`, webpack applies:

```javascript
// Equivalent configuration
module.exports = {
  mode: 'production',
  performance: {
    hints: 'warning',
    maxAssetSize: 250000,
    maxEntrypointSize: 250000
  },
  output: {
    pathinfo: false
  },
  optimization: {
    moduleIds: 'deterministic',
    chunkIds: 'deterministic',
    mangleExports: 'deterministic',
    nodeEnv: 'production',
    flagIncludedChunks: true,
    occurrenceOrder: true,
    concatenateModules: true,
    splitChunks: {
      hidePathInfo: true,
      minSize: 30000,
      maxAsyncRequests: 5,
      maxInitialRequests: 3
    },
    emitOnErrors: false,
    checkWasmTypes: true,
    minimize: true,
    minimizer: [
      new TerserPlugin(/* ... */)
    ],
    removeAvailableModules: true,
    removeEmptyChunks: true,
    mergeDuplicateChunks: true,
    providedExports: true,
    usedExports: true,
    innerGraph: true,
    sideEffects: true
  },
  plugins: [
    new webpack.DefinePlugin({ 'process.env.NODE_ENV': JSON.stringify('production') }),
    new TerserPlugin(),
    new webpack.optimize.ModuleConcatenationPlugin(),
    new webpack.NoEmitOnErrorsPlugin()
  ]
};
```

## DefinePlugin Integration

Mode automatically uses DefinePlugin to set `process.env.NODE_ENV`:

```javascript
// In your code
if (process.env.NODE_ENV === 'production') {
  console.log('Running in production');
} else {
  console.log('Running in development');
}

// After compilation in production mode:
console.log('Running in production');
// Development code is removed (dead code elimination)
```

## Optimization Differences

<CodeGroup>
```javascript Development
// Named module IDs for better debugging
module.id = 'src/components/Button.js';

// Readable chunk names
chunk.name = 'vendors~main';

// Path info in bundle
/*! exports provided: Button, Icon */

// No minification
function myFunction(parameter) {
  return parameter + 1;
}
```

```javascript Production  
// Short deterministic module IDs
module.id = 123;

// Short chunk IDs
chunk.id = 0;

// Minified code
function n(r){return r+1}

// Tree shaking applied
// Unused exports removed
```
</CodeGroup>

## Custom Mode Configuration

You can override defaults while still using a mode:

```javascript webpack.config.js
module.exports = {
  mode: 'production',
  
  // Override production defaults
  optimization: {
    minimize: false, // Disable minification
    splitChunks: {
      chunks: 'all'
    }
  },
  
  // Add source maps in production
  devtool: 'source-map'
};
```

## Environment Variables

<Note>
The mode sets `process.env.NODE_ENV` only within the bundled code, not in webpack configuration files.
</Note>

For configuration files:
```javascript webpack.config.js
module.exports = (env, argv) => {
  const isProduction = argv.mode === 'production';
  
  return {
    mode: argv.mode,
    entry: './src/index.js',
    devtool: isProduction ? 'source-map' : 'eval-source-map',
    plugins: [
      ...(isProduction ? [new BundleAnalyzerPlugin()] : [])
    ]
  };
};
```

## Warnings

<Warning>
If no mode is set, webpack defaults to `'production'` with a warning. Always explicitly set the mode.
</Warning>

From `WarnNoModeSetPlugin.js`, webpack warns when mode is not set:
```javascript
if (!options.mode) {
  compilation.warnings.push(
    new NoModeWarning()
  );
}
```

## Module Concatenation

Production mode enables scope hoisting (module concatenation):

```javascript
// Source modules
// a.js
export const value = 42;

// b.js  
import { value } from './a';
export default value + 1;

// After concatenation in production
(function() {
  const value = 42;
  const b_default = value + 1;
  // ...
})();
```

## Tree Shaking

Production mode enables tree shaking to remove unused exports:

```javascript utils.js
export function used() {
  return 'I am used';
}

export function unused() {
  return 'I am not used';
}
```

```javascript main.js
import { used } from './utils';

console.log(used());
```

In production mode, `unused()` is removed from the bundle.

## Performance Hints

Production mode shows performance warnings:

```javascript
WARNING in asset size limit: The following asset(s) exceed the recommended size limit (244 KiB).
This can impact web performance.
Assets: 
  main.js (512 KiB)

WARNING in entrypoint size limit: The following entrypoint(s) combined asset size exceeds the recommended limit (244 KiB).
Entrypoints:
  main (512 KiB)
    main.js
```

Configure or disable:
```javascript
module.exports = {
  performance: {
    hints: 'warning', // 'error' or false
    maxAssetSize: 500000,
    maxEntrypointSize: 500000
  }
};
```

## None Mode

Use `'none'` when you want full control:

```javascript webpack.config.js
module.exports = {
  mode: 'none',
  optimization: {
    // Manually configure all optimizations
    minimize: true,
    minimizer: [new TerserPlugin()],
    splitChunks: {
      chunks: 'all'
    }
  }
};
```

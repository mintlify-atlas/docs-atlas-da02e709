title: Compilation Hooks
description: Reference for all Compilation hooks available to webpack plugins
---

The `Compilation` module is used by the `Compiler` to create new compilations (or builds). A compilation instance has access to all modules and their dependencies. It orchestrates the application modules, creates the chunk graph, and generates the final assets.

## Overview

Compilation hooks are exposed on the `compilation.hooks` object. Most plugins tap into the `compiler.hooks.compilation` hook to access compilation hooks.

```javascript
compiler.hooks.compilation.tap('MyPlugin', (compilation) => {
  compilation.hooks.optimize.tap('MyPlugin', () => {
    console.log('Assets are being optimized.');
  });
});
```

## Module Hooks

### buildModule

<ParamField path="buildModule" type="SyncHook<[Module]>">

</ParamField>

## Sealing Hooks

The sealing phase happens after all modules have been built and before chunks are created.

### seal

<ParamField path="seal" type="SyncHook<[]>">
  Called when the compilation starts sealing. This is where the chunk graph is created.
</ParamField>

**Parameters:** None

---

### unseal

<ParamField path="unseal" type="SyncHook<[]>">
  Called when the compilation is unsealed (before additional passes).
</ParamField>

**Parameters:** None

---

### beforeChunks

<ParamField path="beforeChunks" type="SyncHook<[]>">
  Called before chunks are created from the module graph.
</ParamField>

**Parameters:** None

---

### afterChunks

<ParamField path="afterChunks" type="SyncHook<[Iterable\<Chunk\>]>">
  Called after chunks have been created. This hook is useful to inspect, analyze, and/or modify the chunk graph.
</ParamField>

**Parameters:**
- `chunks` (Iterable\<Chunk\>) - All chunks in the compilation

**Example:**
```javascript
compilation.hooks.afterChunks.tap('MyPlugin', (chunks) => {
  for (const chunk of chunks) {
    console.log('Chunk:', chunk.name, 'Modules:', chunk.getNumberOfModules());
  }
});
```

### processAssets

<ParamField path="processAssets" type="AsyncSeriesHook<[CompilationAssets]>">
  Called to process assets. This is the main hook for asset optimization.
</ParamField>

**Parameters:**
- `assets` (CompilationAssets) - All assets in the compilation

**Stages:**
- `PROCESS_ASSETS_STAGE_ADDITIONAL` - Add additional assets
- `PROCESS_ASSETS_STAGE_PRE_PROCESS` - Basic preprocessing
- `PROCESS_ASSETS_STAGE_DERIVED` - Derive new assets from existing
- `PROCESS_ASSETS_STAGE_ADDITIONS` - Add additional sections to assets
- `PROCESS_ASSETS_STAGE_OPTIMIZE` - Optimize existing assets
- `PROCESS_ASSETS_STAGE_OPTIMIZE_COUNT` - Optimize asset count
- `PROCESS_ASSETS_STAGE_OPTIMIZE_COMPATIBILITY` - Optimize compatibility
- `PROCESS_ASSETS_STAGE_OPTIMIZE_SIZE` - Optimize size
- `PROCESS_ASSETS_STAGE_DEV_TOOLING` - Add development tooling
- `PROCESS_ASSETS_STAGE_OPTIMIZE_INLINE` - Optimize for inlining
- `PROCESS_ASSETS_STAGE_SUMMARIZE` - Summarize list of assets
- `PROCESS_ASSETS_STAGE_OPTIMIZE_HASH` - Optimize hashes
- `PROCESS_ASSETS_STAGE_OPTIMIZE_TRANSFER` - Optimize transfer
- `PROCESS_ASSETS_STAGE_ANALYSE` - Analyze assets
- `PROCESS_ASSETS_STAGE_REPORT` - Create reports

**Example:**
```javascript
compilation.hooks.processAssets.tapAsync(
  {
    name: 'MyPlugin',
    stage: Compilation.PROCESS_ASSETS_STAGE_OPTIMIZE
  },
  (assets, callback) => {
    Object.keys(assets).forEach(filename => {
      // Process each asset
    });
    callback();
  }
);
```

## Usage Example

```javascript
class MyCompilationPlugin {
  apply(compiler) {
    compiler.hooks.compilation.tap('MyCompilationPlugin', (compilation) => {
      // Access module hooks
      compilation.hooks.buildModule.tap('MyPlugin', (module) => {
        console.log('Building:', module.resource);
      });

      // Access optimization hooks
      compilation.hooks.optimizeChunks.tap('MyPlugin', (chunks) => {
        for (const chunk of chunks) {
          // Optimize chunks
        }
      });

      // Process assets
      compilation.hooks.processAssets.tapAsync(
        {
          name: 'MyPlugin',
          stage: Compilation.PROCESS_ASSETS_STAGE_OPTIMIZE
        },
        (assets, callback) => {
          // Modify assets
          callback();
        }
      );
    });
  }
}
```
